# LMS Platform - Render Deployment Guide

This guide provides step-by-step instructions for deploying the LMS Platform to Render.com.

## Overview

The LMS Platform consists of:
- **Backend**: Laravel 10 API (PHP)
- **Frontend**: React 18 SPA with Vite
- **Database**: MySQL (managed by Render)

## Prerequisites

1. GitHub account with your LMS Platform repository
2. Render.com account (free tier available)
3. All code committed and pushed to your main branch

## Deployment Steps

### 1. Prepare Your Repository

Ensure all the deployment files are in your repository:
- `render.yaml` (root directory)
- `backend/render-build.sh`
- `backend/render-start.sh`
- `backend/.env.production`
- `backend/database/seeders/ProductionSeeder.php`

### 2. Deploy to Render

#### Option A: Using render.yaml (Recommended)

1. Go to [Render Dashboard](https://dashboard.render.com/)
2. Click "New" → "Blueprint"
3. Connect your GitHub repository
4. Select the repository containing your LMS Platform
5. Click "Connect"
6. Render will automatically detect the `render.yaml` file and create all services

#### Option B: Manual Deployment

If you prefer to deploy services manually:

##### Deploy Backend (Laravel API)

1. Go to [Render Dashboard](https://dashboard.render.com/)
2. Click "New" → "Web Service"
3. Connect your GitHub repository
4. Configure:
   - **Name**: `lms-backend`
   - **Root Directory**: `backend`
   - **Runtime**: `PHP`
   - **Build Command**: `./render-build.sh`
   - **Start Command**: `./render-start.sh`
   - **Plan**: Free
5. Add Environment Variables (see section below)
6. Click "Create Web Service"

##### Deploy Frontend (React App)

1. Click "New" → "Static Site"
2. Connect your GitHub repository
3. Configure:
   - **Name**: `lms-frontend`
   - **Root Directory**: `frontend`
   - **Build Command**: `npm ci && npm run build`
   - **Publish Directory**: `dist`
   - **Plan**: Free
4. Add Environment Variables:
   - `VITE_API_URL`: URL of your backend service (e.g., `https://lms-backend.onrender.com`)
5. Click "Create Static Site"

##### Create Database

1. Click "New" → "PostgreSQL" or "MySQL"
2. Configure:
   - **Name**: `lms-database`
   - **Database Name**: `lms_platform`
   - **Username**: `lms_user`
   - **Plan**: Free
3. Click "Create Database"

### 3. Environment Variables

#### Backend Environment Variables

Set these in your backend service:

**Required Variables:**
- `APP_NAME`: "LMS Platform"
- `APP_ENV`: production
- `APP_DEBUG`: false
- `APP_URL`: (auto-generated by Render)
- `DB_HOST`: (auto-generated by Render)
- `DB_PORT`: (auto-generated by Render)
- `DB_DATABASE`: (auto-generated by Render)
- `DB_USERNAME`: (auto-generated by Render)
- `DB_PASSWORD`: (auto-generated by Render)

**Auto-generated Variables:**
- `APP_KEY`: (generated by Render)
- `JWT_SECRET`: (generated by Render)

**Optional Variables (for enhanced security):**
- `ADMIN_PASSWORD`: Strong password for admin user
- `INSTRUCTOR_PASSWORD`: Strong password for instructor user
- `STUDENT_PASSWORD`: Strong password for student user

#### Frontend Environment Variables

Set these in your frontend service:
- `VITE_API_URL`: URL of your backend service (e.g., `https://lms-backend.onrender.com`)

### 4. Custom Domain (Optional)

1. In your Render service dashboard, go to "Settings"
2. Scroll to "Custom Domains"
3. Add your domain
4. Update DNS records as instructed by Render

### 5. SSL Certificate

Render automatically provides SSL certificates for all services. Your app will be available over HTTPS.

## Post-Deployment

### 1. Verify Deployment

1. **Backend Health Check**: Visit `https://your-backend-url.onrender.com/api/test`
   - Should return: `{"message": "API is working!"}`

2. **Frontend Access**: Visit your frontend URL
   - Should load the LMS Platform interface

### 2. Initial Setup

1. Access your frontend application
2. Login with the admin credentials:
   - Email: `admin@lmsplatform.com`
   - Password: Value of `ADMIN_PASSWORD` or `AdminPass123!`

3. Change default passwords immediately
4. Configure your LMS settings as needed

### 3. Monitor Services

- Check Render dashboard for service health
- Monitor logs for any issues
- Set up monitoring alerts if needed

## Troubleshooting

### Common Issues

1. **Build Failures**
   - Check build logs in Render dashboard
   - Ensure all dependencies are properly specified
   - Verify file permissions for shell scripts

2. **Database Connection Issues**
   - Verify environment variables are correctly set
   - Check database service status
   - Ensure migrations ran successfully

3. **API Connection Issues**
   - Verify `VITE_API_URL` is correctly set
   - Check CORS settings in Laravel
   - Ensure backend service is running

### Getting Help

1. Check Render documentation: https://render.com/docs
2. Review Laravel deployment guides
3. Check application logs in Render dashboard

## Scaling

### Free Tier Limitations

- Services sleep after 15 minutes of inactivity
- Limited to 750 hours per month
- 100GB bandwidth per month

### Upgrading

Consider upgrading to paid plans for:
- Always-on services
- Better performance
- More resources
- Enhanced support

## Security Considerations

1. **Environment Variables**: Never commit sensitive data to Git
2. **Database Access**: Use strong passwords and restrict access
3. **HTTPS**: Always use HTTPS in production (enabled by default on Render)
4. **Regular Updates**: Keep dependencies updated
5. **Monitoring**: Set up logging and monitoring

## Maintenance

### Regular Tasks

1. Monitor application performance
2. Review security updates
3. Backup database regularly
4. Update dependencies as needed
5. Monitor usage and costs

### Updates

1. Push code changes to your main branch
2. Render will automatically redeploy
3. Test thoroughly after each deployment
